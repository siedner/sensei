# Port Context

Use this contextual overlay for Port and developer-platform exercises:

**Persona → workflow → context → action → guardrail → interface → metric**

Potential personas include platform engineer, developer, engineering manager, SRE, infrastructure owner, security owner, AI-agent governance owner, and internal developer portal owner.

Relevant primitives include software catalog, engineering context, blueprints, self-service actions, workflow orchestration, scorecards, dashboards, AI agents, access control, approvals, audit logs, integrations, API, CLI, chat, and coding-agent surfaces.

Always distinguish passive visibility from a product that enables action. Name the human approval boundary, permissions, audit evidence, and rollback path.

Do not invent company facts or interviewer expectations.
