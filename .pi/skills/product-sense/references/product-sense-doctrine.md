# Product-Sense Doctrine

Strong product sense is judgment, not feature brainstorming:

1. Establish the objective, assumptions, and why-now rationale.
2. Map relevant actors, then choose one distinct target segment.
3. Walk the workflow and diagnose one behavioral, specific, consequential pain.
4. Prioritize one problem with an explicit rationale.
5. Generate three genuinely different mechanisms.
6. Choose a focused v1 and state exclusions.
7. Explain the key tradeoff and failure modes.
8. Connect user value to behavior, metrics, and guardrails.
9. Close without adding new ideas.

A need is a desired outcome. A problem blocks that outcome. A solution is an intervention. Do not confuse them.
